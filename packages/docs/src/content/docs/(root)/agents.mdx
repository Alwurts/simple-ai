## How It Works

When you chat with an agent:

1. **You mention an agent** in your message (like "@weather-agent")
2. **The system routes** your message to the appropriate agent
3. **The agent thinks** about your request and decides what to do
4. **Tools are called** automatically (like fetching weather data)
5. **Response streams back** in real-time to your chat

Agents combine AI reasoning with practical tools to give you helpful, accurate answers.

## Technical Reference

### Agent Structure

Each agent consists of:

```typescript
// Agent Definition
export const myAgent = new Agent({
  model: model.languageModel("gpt-4"),
  tools: { "my-tool": myTool },
  stopWhen: stepCountIs(20),
});

// System Prompt
export const myAgentPrompt = `You are a helpful assistant...`;

// Tool Implementation
export const myTool = tool({
  description: "Tool description",
  parameters: z.object({ input: z.string() }),
  execute: async ({ input }) => { /* logic */ },
});
```

### Agent Execution Flow

1. User mentions agent in message
2. System routes to appropriate agent
3. Agent analyzes query and decides on tools
4. Tools execute and return data
5. Agent formulates response with tool results
6. Response streams back to UI

### API Integration

The chat-04 block handles all the complex routing and streaming automatically. Your API route simply needs to handle the agent execution:

```typescript
// app/api/chat/route.ts
import { agentExecute, agentRoute } from "@/lib/ai/agent-route-respond";

export async function POST(req: Request) {
  const { messages, mentions } = await req.json();
  const agentId = agentRoute(mentions);
  return agentExecute(agentId, messages);
}
```

### Best Practices

- **Clear system prompts**: Define agent roles and limitations
- **Focused tools**: Give agents only necessary tools
- **Reasonable step limits**: Prevent infinite loops
- **Error handling**: Tools should handle failures gracefully
