## Overview

The Resizable Node is a foundational React Flow node component that adds resizing capabilities to your nodes. Built on top of React Flow's NodeResizer, it provides a clean interface with size constraints and hover effects. This component serves as a base for other resizable nodes in the application.

## Usage

The Resizable Node provides a wrapper component that adds resizing functionality to your nodes. Here's how to use it in your React Flow application:

```tsx
import { ResizableNode } from "@/components/ui/flow/resizable-node"
import { type NodeProps } from "@xyflow/react"

function CustomNode(props: NodeProps) {
  return (
    <ResizableNode
      selected={props.selected}
      className="flex items-center justify-center p-4"
    >
      <div>Your node content here</div>
    </ResizableNode>
  )
}

// Use in your React Flow instance
const nodeTypes = {
  "custom-node": CustomNode
}
```

### Features

The node includes several key features:

1. **Size Constraints**
   - Minimum width: 250px
   - Minimum height: 200px
   - Maximum width: 800px
   - Maximum height: 800px

2. **Resize Handles**
   - Visible when node is selected
   - Smooth resizing interaction
   - Maintains aspect ratio (optional)

3. **Styling**
   - Clean base styling
   - Orange highlight on hover
   - Customizable through className prop

4. **Base Node Integration**
   - Built on top of BaseNode component
   - Maintains consistent node styling
   - Compatible with React Flow's node system

## Props

| Prop | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| selected | boolean | Yes | - | Whether the node is selected (controls resize handles visibility) |
| className | string | No | - | Additional CSS classes to apply to the node |
| style | React.CSSProperties | No | - | Additional inline styles (size constraints cannot be overridden) |
| children | React.ReactNode | No | - | The content to render inside the node |

### Size Constraints

The ResizableNode comes with built-in size constraints that can be overridden:

```tsx
{
  minHeight: 200,
  minWidth: 250,
  maxHeight: 800,
  maxWidth: 800
}
```

These constraints ensure that nodes maintain a usable size while preventing them from becoming too large.
